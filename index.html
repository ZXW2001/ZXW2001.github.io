linux下终端代理设置

linux系统设置中代理无法作用于终端中运行的程序，因此需要一个代理链工具来完成转发。Proxychains 是linux下一个代理链工具，只要在配置文件中写入代理服务器信息，在希望走代理的命令前加入proxychains，其后的指令就会自动从设置的代理运行。

工具下载

​ sudo apt install proxychains

工具配置

proxychains所有配置均在**/etc/proxychains.conf **中

vim /etc/proxychains.conf

第一段为代理链接选择的方式，Proxychains支持同时配置多个代理，在使用是会自主。此处有三个选项：严格代理，随机代理，动态代理;一般选择动态。

在文件末尾有代理链接的设置

格式为

协议 地址 端口

这些信息可以在所使用的代理软件中找到

以QV2ray为例，设置中包含了http/socks5两种协议的端口，选择其中一种即可。

可能问题

引用地址：https://parrotsec-cn.org/t/proxychains/3012

本来以为很好解决，首先修改的 /usr/bin/proxychains
奈何这样做之后，只有用 sudo proxychains 的时候才可以 curl www.google.com ，普通用户依旧是提示上面那个错误
重现错误
今天测试一个公司网站的时候，用脚本去扫xss没想到遇到一个脸书的鬼链接，本着扶墙的原则，那就扶墙扫，之后就出现了上述错误，但是我已经修改了 /usr/bin/proxychains 为啥还会报错
遗留已久
这个小问题恶心我好久了，然后我就直接去谷歌了，认真查了之后，果然发现了，不能只修改 /usr/bin/proxychains ，还得修改 /usr/lib/proxychains3/proxyresolv 才行

彻底解决
修改 /usr/bin/proxychains
sudo vi /usr/bin/proxychains
export LD_PRELOAD=libproxychains.so.3
改为
export LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libproxychains.so.3
修改 /usr/lib/proxychains3/proxyresolv
sudo vi /usr/lib/proxychains3/proxyresolv
export LD_PRELOAD=libproxychains.so.3
改为
export LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libproxychains.so.3
之后普通用户再使用 Proxychains 舒服得一皮

工具使用

alias

为方便使用，可以给proxychains加一条alias，毕竟敲这么长单词很不爽

打开.zshrc(或者.bashrc)，在文件末添加

​ alias pc=”proxychains”

以后在使用时只需要在文件开头添加pc 即可

zsh插件

为方便使用，编写了一个简单的插件，可以快速向文件头补充proxychains指令。

项目地址： https://github.com/ZXW2001/proxychains

在安装zsh后按教程安装插件。
